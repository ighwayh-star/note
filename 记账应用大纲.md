# 内置 AI Agent 的记账应用：项目大纲（Vue + FastAPI + LangChain）

## 1. 目标与范围

### 1.1 核心能力

- 自然语言记账：用户直接发“昨天星巴克 38”“今天地铁 4 块”，AI 自动生成并写入流水
- 自然语言改/删：用户发“把刚才那笔改成分类：咖啡”“删掉今天早上那笔早餐”，AI 自主定位并修改/删除
- 对话式查询与统计：用户问“我上个月餐饮花了多少”“今年房租趋势”“本周预算还剩多少”，AI 返回可视化友好的结构化结果
- 手动 CRUD：支持传统表单录入、列表编辑、分类/账户/预算管理、导入导出

### 1.2 关键原则

- AI 只能通过后端“受控工具”读写数据（Tool Calling），不允许拼接 SQL 或直接执行任意代码
- 所有 AI 写操作必须可追溯（审计日志）与可撤销（回滚/软删除/事件记录）
- 高风险操作二次确认：删除、批量修改、导入覆盖、跨期大额调整等

## 2. 总体架构（推荐）

### 2.1 前端（Web 与 App 共用）

- Vue 3 + TypeScript + Vite
- 状态管理：Pinia
- 路由：Vue Router
- UI：Element Plus / Naive UI（二选一，按团队偏好）
- 图表：ECharts / Chart.js（二选一）

### 2.2 后端

- FastAPI + Pydantic
- ORM：SQLAlchemy（或 SQLModel）
- 迁移：Alembic
- 数据库：PostgreSQL
- 缓存/队列：Redis
- 异步任务：Celery / RQ（二选一）

### 2.3 AI 接入（LangChain）

- LangChain（Python）作为 Agent 编排层
- LLM 提供方可插拔（OpenAI/Anthropic/本地模型/云厂商），通过 LangChain 的 ChatModel 接口统一
- 可观测：LangSmith（可选）用于链路追踪、提示词与工具调用调试

## 3. 数据模型与业务设计（后端优先定）

### 3.1 核心实体

- users：用户与偏好（币种、时区、预算规则）
- accounts：账户（现金/银行卡/信用卡/第三方支付）
- categories：分类（收入/支出、多级）
- transactions：流水（金额、方向、时间、分类、账户、商户、标签、备注、创建来源：AI/手动）
- budgets：预算（按月/周/自定义周期，支持分类维度）
- audit_logs：所有写操作记录（来源、工具参数、前后差异、操作者：AI/用户、请求标识）

### 3.2 关键业务能力

- 金额与币种：统一精度与舍入规则；支持多币种时的汇率策略（可后续迭代）
- 时间归一：时区处理；自然语言日期“昨天/上周五/本月初”解析后落库为标准时间
- 幂等与去重：防止重复记账（时间窗口 + 金额 + 商户/备注相似度）
- 撤销与回滚：基于审计日志实现最近 N 次 AI 操作撤销（或事件溯源）

## 4. LangChain Agent 设计（让 AI 能改数据但可控）

### 4.1 交互输入与输出

- 输入：自然语言 +（可选）上下文信息（当前筛选条件、最近一笔、用户时区/币种）
- 输出：自然语言解释 + 结构化结果（用于前端渲染卡片/列表/图表）+ 执行动作摘要（用于审计与撤销）

### 4.2 Agent 形态（推荐路线）

- 方案 A（推荐）：工具型 Agent（Tool Calling）
  - LangChain Agent 负责意图识别与工具选择
  - 业务写入由后端工具函数完成，工具函数做权限、校验、幂等等
  - 适合“能自主改数据”的强诉求
- 方案 B：Structured Extraction + 规则路由
  - 先用 LangChain 做结构化抽取（金额/时间/分类/账户/方向）
  - 再用确定性规则决定调用哪个接口/服务
  - 可作为 MVP 阶段降低不可控性

### 4.3 工具（Tools）清单（示例）

- 写操作
  - create_transaction(amount, currency, direction, occurred_at, category_id?, account_id?, merchant?, note?, tags?)
  - update_transaction(transaction_id or locator, patch)
  - delete_transaction(transaction_id or locator, mode=soft)
  - upsert_category(name, parent_id?, type)
  - upsert_budget(period, category_id?, amount)
- 读操作
  - search_transactions(query, filters, page, page_size)
  - get_stats(range, group_by=[category/account/day], direction, top_k)
  - get_budget_status(period)
- 辅助
  - suggest_category(merchant, note, amount, direction)
  - resolve_locator(locator) 通过“最近一笔/上周那笔/今天早餐”等定位候选流水并给出置信度

### 4.4 受控写入与防护（强制）

- 所有工具入参由 Pydantic 校验；不可信字段做白名单与长度限制
- 工具内部始终以 user_id 为主键隔离数据，避免越权读取/修改
- 结果限制：查询最多返回 N 条；统计时间范围最大跨度；防止 prompt 注入导致大范围导出
- 二次确认策略
  - 删除/批量修改：必须返回候选列表并让用户确认目标
  - locator 命中多条：返回候选让用户选中或追加约束

### 4.5 记忆与检索（可选增强）

- 目标：提高分类建议与商户归类准确率、理解用户偏好
- 方案
  - 轻量：基于历史统计的规则/朴素贝叶斯
  - 向量检索：pgvector（PostgreSQL 扩展）或 Qdrant
    - 向量化对象：商户名、备注、标签、分类路径
    - 检索用于 suggest_category / resolve_locator 的候选召回

### 4.6 可观测与评估（推荐）

- LangSmith（可选）：记录每次对话的链路、提示词版本、工具调用、模型输出
- 离线评测集：常见记账语句、边界输入（含歧义时间/金额/分类），用于回归

## 5. 后端开发大纲（FastAPI）

### 5.1 项目结构（建议）

- api：路由（auth/transactions/categories/accounts/budgets/stats/agent）
- core：配置、日志、依赖注入、安全（鉴权、限流）
- models：ORM 模型
- schemas：Pydantic 请求/响应模型
- services：业务服务（记账、统计、预算、导入导出）
- agent：LangChain 编排与工具注册（只做编排，不直接写库）
- workers：异步任务（导入、embedding 更新、定时报表）

### 5.2 API 设计要点

- Auth：注册/登录/刷新 token/退出/设备管理
- CRUD：transactions/categories/accounts/budgets 标准增删改查
- Stats：趋势、分类占比、Top 商户、预算执行
- Agent：POST /agent/chat
  - request：message + context（可选）
  - response：reply_text + actions + structured_payload（用于 UI）

### 5.3 统计与性能

- 索引：user_id + occurred_at、category_id、account_id；必要时加组合索引
- 聚合策略：常用维度可做按月汇总表或缓存；避免每次对话全表聚合
- 并发控制：写入幂等键 + 事务；防止重复提交

### 5.4 测试与质量

- 单元测试：服务层（交易写入、幂等、预算扣减）
- 集成测试：API（鉴权、分页、过滤、统计一致性）
- Agent 回归：工具调用路径稳定性、边界输入（歧义/缺字段）处理

## 6. 前端开发大纲（Vue）

### 6.1 页面与模块

- 登录/注册/设置（币种、时区、默认账户）
- 首页仪表盘：本月收支、预算剩余、趋势图
- 流水列表：筛选、搜索、编辑、批量操作、导出
- 手动记一笔：金额键盘、分类选择、账户选择、备注与标签
- AI 对话页
  - 聊天气泡
  - AI 执行动作卡片（新增/修改/删除摘要）
  - 撤销入口与候选选择（当 AI 需要确认目标时）
- 分类/账户/预算管理
- 导入导出：CSV（MVP），后续可加 OFX/支付宝微信账单解析

### 6.2 交互与体验要点

- AI 返回结构化 payload：列表/图表直接渲染，而非仅显示文本
- 写操作的可控反馈：展示“将新增一笔/将修改哪一笔”，必要时让用户确认
- 撤销能力：对最近一次 AI action 提供一键撤销

### 6.3 工程化

- 请求层：Axios 封装、统一错误码、token 刷新
- 权限守卫：路由拦截与会话恢复
- 可观测：前端错误上报（可选 Sentry）

## 7. 部署与上线（网页端 + 手机端）

### 7.1 环境与配置

- dev/staging/prod 三套环境
- 机密配置：数据库连接、JWT 密钥、LLM Key、LangSmith Key（如用）均用环境变量与密钥管理

### 7.2 容器化与运行形态

- Docker：api、worker、postgres、redis、nginx
- 线上编排
  - 小规模：Docker Compose
  - 中大型：Kubernetes（Ingress、HPA、滚动发布）

### 7.3 Web 发布

- 前端静态资源：CDN 或 Nginx
- 后端：HTTPS、CORS、速率限制、WAF（可选）
- 数据库：定时备份 + 恢复演练

### 7.4 手机端发布路线

- 路线 A：PWA（最快上线，免商店）
- 路线 B（推荐）：Capacitor 打包 iOS/Android
  - 复用 Vue 代码
  - 支持更好的推送、文件访问、生物识别等原生能力

### 7.5 CI/CD 与运维

- CI：测试、构建、镜像推送、静态检查（lint/typecheck）
- 监控：日志、指标、告警；错误收集（如 Sentry）
- 审计合规：保留关键写操作链路（不保存敏感原文或进行脱敏）

## 8. 里程碑（建议顺序）

- MVP
  - 用户体系 + 手动记账 CRUD
  - 基础统计（按月收支、分类占比）
- AI v1（LangChain 接入）
  - 自然语言记一笔、查统计（受控工具 + 审计）
  - locator 仅支持“最近一笔/指定日期金额”这类确定性定位
- AI v2
  - 改/删/批量操作 + 二次确认
  - 分类建议与去重增强（规则 + 向量检索可选）
  - 撤销与回滚完善
- 上线与扩展
  - Web 正式发布 + PWA/Capacitor 上架
  - 监控、备份、灰度、评测集与提示词版本管理
