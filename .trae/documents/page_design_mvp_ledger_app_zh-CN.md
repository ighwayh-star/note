# 页面设计文档（Desktop-first）

## 0. Global Styles（全局）
- 设计基调：简洁、可读、输入效率优先。
- 布局策略：桌面端固定内容宽度（max-width: 1100px）居中；移动端单列自适应。
- Breakpoints：
  - Desktop：≥1024px
  - Tablet：768–1023px
  - Mobile：<768px
- 颜色 Token（示例，可在实现中落地为 CSS 变量）：
  - `--bg`: #F7F8FA（页面背景）
  - `--card`: #FFFFFF（卡片背景）
  - `--text`: #111827（主文字）
  - `--muted`: #6B7280（次要文字）
  - `--primary`: #2563EB（主按钮）
  - `--danger`: #DC2626（删除/危险操作）
  - `--border`: #E5E7EB（分割线）
- 字体与字号：
  - 字体：系统字体栈（优先）；数字金额可启用等宽数字（tabular-nums）。
  - H1 20–24px，H2 16–18px，正文 14–16px，辅助文字 12–13px。
- 组件基线：
  - Button：Primary/Secondary/Danger；hover 提升阴影或加深底色；disabled 降低透明度。
  - Input：默认 40px 高；错误态显示红色边框 + 错误文案。
  - Card：圆角 10–12px；轻阴影；内部 padding 16–20px。
  - Modal/Drawer：桌面端 Modal 居中；移动端优先 Drawer 从底部弹出。

## 1. 登录/注册页

### 1.1 Layout
- 使用 Flexbox 居中布局：页面全高（min-height: 100vh），卡片居中。
- Desktop：卡片宽 420–480px；Mobile：左右留白 16px，卡片撑满。

### 1.2 Meta Information
- Title：登录 / 注册 - 记账
- Description：登录后开始记录你的收支。
- Open Graph：
  - `og:title` = 登录 / 注册 - 记账
  - `og:description` = 登录后开始记录你的收支。

### 1.3 Page Structure
- 顶部：产品名 + 一句话说明。
- 中部：登录/注册 Tab + 表单。
- 底部：隐私/条款占位链接（仅 UI，不扩展功能）。

### 1.4 Sections & Components
1. 认证卡片（Auth Card）
   - Tab：`登录` / `注册`。
   - 字段：邮箱、密码。
   - CTA：
     - 登录态按钮文案“登录并进入”。
     - 注册态按钮文案“创建账号”。
   - 错误提示：
     - 表单级错误（如账号密码错误）置顶展示。
     - 字段级校验（邮箱格式、必填）。
2. Loading 状态
   - 提交中按钮显示 loading；表单禁用防重复提交。

## 2. 首页（仪表盘）

### 2.1 Layout
- Desktop：CSS Grid 两列布局
  - 左列（主内容，约 2/3）：统计摘要 + 近期流水
  - 右列（侧栏，约 1/3）：快速记一笔 + 时间范围切换
- Tablet/Mobile：折叠为单列，快速记一笔置顶，其次摘要，再近期流水。

### 2.2 Meta Information
- Title：仪表盘 - 记账
- Description：快速记一笔，查看本期收支摘要。
- Open Graph 同步 Title/Description。

### 2.3 Page Structure
- 顶部：全局导航栏（粘性可选）。
- 内容区：两列网格卡片区。

### 2.4 Sections & Components
1. 顶部导航栏（Top Nav）
   - 左侧：Logo/产品名。
   - 中/右：导航链接（首页、明细）。
   - 右侧：用户菜单（显示邮箱缩写/头像占位）+ 退出登录。
2. 时间范围切换（Range Switch）
   - 默认：本月。
   - 选项：本月 / 上月 / 自定义（自定义为开始/结束日期选择）。
   - 交互：切换后立即刷新摘要与近期流水。
3. 快速记一笔（Quick Add Form）
   - 表单字段：类型（Segmented Control：收入/支出）、金额（Number Input）、日期（Date Picker，默认今天）、备注（可选）。
   - CTA：“保存”。
   - 校验：金额 > 0；日期必填。
   - 成功反馈：轻提示（toast）+ 表单清空或保留金额（实现可选）。
4. 统计摘要卡（Summary Cards）
   - 指标：总收入、总支出、结余、笔数。
   - 金额展示：大号数字 + 次要说明（所选时间范围）。
   - 可视化（可选）：小型柱状/折线（迷你图）用于趋势占位，不新增统计口径。
5. 近期流水（Recent Entries）
   - 列表项：类型徽标（收入/支出）、金额（颜色区分：收入主色/支出危险色或中性）、日期、备注（最多两行省略）。
   - 操作：
     - “查看全部”跳转到明细页并带入当前时间范围。
     - 点击单条可跳转明细页并定位/预筛选（实现方式按路由 query）。

## 3. 收支明细页

### 3.1 Layout
- Desktop：上方工具栏 + 下方列表区域。
  - 工具栏：筛选器在左，新增按钮在右。
  - 列表：表格风格（可排序仅限时间倒序固定，不新增功能），或卡片列表（二选一）。
- Mobile：筛选器折叠为“筛选”按钮弹出 Drawer；列表为卡片。

### 3.2 Meta Information
- Title：收支明细 - 记账
- Description：查看并管理你的收支记录。

### 3.3 Page Structure
- 顶部：与首页一致的全局导航栏。
- 工具栏：类型筛选 + 日期区间 + 重置。
- 主体：列表 + 分页/加载更多。

### 3.4 Sections & Components
1. 筛选工具栏（Filter Bar）
   - 类型：全部/收入/支出。
   - 日期区间：开始日期、结束日期。
   - 操作：应用筛选（可自动触发）+ 重置。
2. 新增按钮（New Entry）
   - 打开新增弹窗/抽屉（与首页字段一致）。
3. 明细列表（Entries List）
   - 列：日期、类型、金额、备注、操作。
   - 空态：提示“暂无记录”，引导新增。
   - Loading：骨架屏或 loading 行。
4. 编辑/删除
   - 编辑：打开与新增同款表单，预填数据。
   - 删除：二次确认弹窗；确认后显示 loading 并从列表移除。
5. 分页/加载更多
   - Desktop：分页组件（上一页/下一页/页码）。
   - Mobile：加载更多按钮（或无限滚动，占位实现）。

## 4. 通用交互与状态（跨页面）
- 权限控制：未登录访问首页/明细时，跳转登录页；登录成功回跳原目标页。
- 错误处理：
  - 401：提示“登录已过期”，跳转登录。
  - 网络错误：保留当前表单内容，提示重试。
- 可用性：所有金额输入支持键盘快速提交（Enter 触发保存）。
- 动效：按钮 hover/按下微交互；Modal/Drawer 使用 150–250ms 过渡。
